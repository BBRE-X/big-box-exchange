# BIG BOX EXCHANGE — PRODUCT RULES

---

## CORE THESIS

This is NOT a listings platform.

This is a:
→ DEMAND-LED COMMERCIAL REAL ESTATE EXCHANGE

---

## PRIMARY OBJECT

Acquisition Mandate = CORE OBJECT

Everything revolves around:
- buyers posting requirements
- market responding to demand

---

## SECONDARY OBJECTS

Assets:
- supply side
- optional visibility
- can respond to mandates

Deal Rooms:
- execution layer
- contain deals
- track pipeline stages

---

## ACCOUNT STRUCTURE

- Company-based accounts (NOT individuals)
- Users belong to companies via memberships
- Roles control permissions

---

## KEY RULES

1. NO duplicate assets submitted to same mandate
2. Messaging only after acceptance
3. Submissions must be structured (no spam)
4. Visibility must be controlled:
   - public
   - restricted
   - private

---

## MATCHING LOGIC

Matches based on:
- location
- asset type
- price range
- size

Must include:
→ “Why this matched” explanation

---

## DEAL ROOM LOGIC

Deal Room = container  
Deals = individual opportunities inside room  

Each deal has:
- stage
- notes
- participants

---

## MONETISATION (DO NOT BREAK)

- subscription access
- pay-per-mandate
- verification tiers
- service provider upsell

---

## UX PRINCIPLES

- Clean
- Fast
- Trust-driven
- No clutter
- Professional (institutional level)

---

## MVP SCOPE

IN:
- mandates
- assets
- deal rooms
- matching
- messaging (gated)

OUT:
- no residential
- no social features
- no unnecessary dashboards

---

## FINAL RULE

Every feature must answer:
→ Does this improve deal flow?

If not, do not build it.
