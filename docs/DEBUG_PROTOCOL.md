# DEBUG PROTOCOL

When something fails:

---

## STEP 1 — IDENTIFY ERROR

- Exact error message
- Where it occurs
- When it occurs

---

## STEP 2 — TRACE SOURCE

- What file triggered it?
- What function?
- What data is involved?

---

## STEP 3 — CHECK:

- Database schema
- RLS policies
- API calls
- Component props

---

## STEP 4 — ROOT CAUSE

Explain clearly:
→ WHY the error is happening

---

## STEP 5 — FIX

- Minimal fix
- Clean fix
- No hacks

---

## STEP 6 — VERIFY

- What should happen now?
- How to test?

---

## RULES

❌ No guessing  
❌ No patch fixes  
❌ No ignoring warnings  

✅ Always explain  
✅ Always verify
